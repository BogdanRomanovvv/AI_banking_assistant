# Промт для улучшения UI/UX дизайна системы Banking AI Assistant

## Контекст проекта
Это веб-приложение для автоматизированной обработки корреспонденции банка. Пользователи: операторы, юристы, маркетологи, администраторы. Основные экраны: Kanban-доска писем, детальный просмотр письма, дашборд аналитики, управление пользователями.

## Цель улучшения
Модернизировать визуальный дизайн приложения, улучшить UX взаимодействия, сделать интерфейс более профессиональным и удобным для ежедневной работы.

---

## Требования к улучшению

### 1. Типографика и шрифты
**Текущее состояние:** Используется Inter с fallback на системные шрифты
**Требования:**
- Подключить современный профессиональный шрифт (например, **Inter**, **Manrope**, **IBM Plex Sans**, **Source Sans Pro**)
- Использовать шрифт с переменной толщиной (variable font) для гибкости
- Настроить иерархию размеров:
  - Заголовки: 20-24px (bold/semibold)
  - Подзаголовки: 16-18px (semibold)
  - Основной текст: 14-15px (regular/medium)
  - Мелкий текст (метки, бейджи): 12-13px (medium)
- Улучшить межстрочный интервал (line-height) для читаемости: 1.5-1.6 для текста, 1.2-1.3 для заголовков
- Добавить более контрастные цвета текста для лучшей читаемости

### 2. Кнопки (Buttons)
**Текущие проблемы:** Стандартные кнопки, недостаточно визуальной иерархии

**Требования к улучшению:**
- **Размеры кнопок:**
  - Large: 48px высота, padding 16-24px
  - Medium (по умолчанию): 40px высота, padding 12-20px
  - Small: 32px высота, padding 8-16px
  
- **Визуальные состояния:**
  - Hover: плавное затемнение/осветление, поднятие (transform), увеличение тени
  - Active: вдавливание (transform), уменьшение тени
  - Disabled: opacity 0.5, cursor not-allowed
  - Loading: спиннер внутри кнопки с сохранением размера
  
- **Типы кнопок:**
  - Primary: яркий акцент (например, #2563EB blue-600), для главных действий
  - Secondary: нейтральный серый или outline, для вторичных действий
  - Success: зеленый (#16A34A green-600), для подтверждений
  - Danger: красный (#DC2626 red-600), для удалений/отмен
  - Ghost/Text: прозрачный фон, только текст с hover-эффектом
  
- **Иконки в кнопках:**
  - Использовать иконки из библиотеки (Lucide, Heroicons, Phosphor)
  - Размер иконок: 16-20px
  - Отступ между иконкой и текстом: 8px
  - Поддержка icon-only кнопок (квадратные, с padding равным по всем сторонам)

- **Скругления:** border-radius 6-8px (современный, не слишком круглый)

### 3. Kanban Board (Доска писем)
**Текущие проблемы:** Базовая визуализация, недостаточно пространства

**Требования:**
- **Колонки:**
  - Ширина: 320-360px (вместо текущих 300px)
  - Фон колонок: светло-серый (#F1F5F9 slate-100)
  - Заголовок колонки: более крупный, с иконкой статуса
  - Добавить тень при drag-over для визуальной обратной связи
  
- **Карточки писем:**
  - Увеличить внутренние отступы: 16px (вместо 12px)
  - Добавить акцентную цветную полоску слева в зависимости от приоритета
  - Hover: плавное поднятие (transform: translateY(-2px)), увеличение тени
  - При перетаскивании: уменьшить opacity до 0.6, добавить пунктирную рамку
  - Выделенная карточка: синяя рамка 2px, фон слегка окрашен
  
- **Бейджи приоритета:**
  - Высокий: красный фон (#FEE2E2 red-100), красный текст (#991B1B red-800)
  - Средний: желтый фон (#FEF3C7 amber-100), темно-желтый текст (#92400E amber-800)
  - Низкий: зеленый фон (#D1FAE5 green-100), темно-зеленый текст (#065F46 green-800)
  - Добавить иконку перед текстом (например, ●)

- **Анимации:**
  - Плавное появление карточек при загрузке (fade-in + slide-up)
  - Smooth drag-and-drop с визуальным feedback
  - Transition на всех hover/active состояниях: 200-300ms

### 4. Модальные окна (Letter Detail, Forms)
**Требования:**
- **Размеры:**
  - Максимальная ширина: 900-1000px
  - Максимальная высота: 90vh с overflow-y: auto
  - Отступы: 32px padding внутри
  
- **Структура:**
  - Четкое визуальное разделение секций (серый фон или border)
  - Заголовки секций: uppercase, letter-spacing, меньший размер, серый цвет
  - Контент секций: белый фон, border-radius, shadow
  
- **Формы:**
  - Инпуты: высота 44px, padding 12-16px, border 2px
  - Focus state: синяя рамка, box-shadow для glow-эффекта
  - Labels: над инпутом, font-weight 600, margin-bottom 8px
  - Error state: красная рамка, красный текст ошибки под полем
  
- **Табы (для вариантов ответов):**
  - Горизонтальные табы с нижним подчеркиванием активного таба
  - Hover: изменение цвета текста и фона
  - Active: синее подчеркивание 3px, синий текст

### 5. Dashboard (Аналитика)
**Требования:**
- **Метрические карточки:**
  - Grid layout: 4-5 карточек в ряд на больших экранах
  - Padding: 24px
  - Большие цифры: 36-40px, bold, цветные (синий для нейтральных, зеленый для позитивных, красный для негативных)
  - Иконки рядом с метриками (диаграммы, стрелки вверх/вниз)
  
- **Графики:**
  - Использовать мягкие цвета: синий (#3B82F6), зеленый (#10B981), оранжевый (#F59E0B)
  - Добавить grid lines для читаемости
  - Tooltip при hover на точках/барах
  
- **Прогресс-бары:**
  - Высота: 8-10px
  - Скругленные края
  - Градиенты или solid цвета в зависимости от значения
  - Анимация заполнения при загрузке страницы

### 6. Header (Шапка приложения)
**Требования:**
- Высота: 64px
- Sticky position с backdrop-blur эффектом при скролле
- Логотип: иконка + текст, выразительный
- Навигация: четкие кнопки/ссылки с hover-эффектами
- Информация о пользователе: аватар (circle, 40px), имя, роль
- Кнопка выхода: иконка + текст, danger-стиль при hover

### 7. Цветовая палитра
**Рекомендации:**
- **Primary (синий):** #2563EB → #1D4ED8 (blue-600 → blue-700)
- **Success (зеленый):** #16A34A → #15803D (green-600 → green-700)
- **Warning (желтый/оранжевый):** #F59E0B → #D97706 (amber-500 → amber-600)
- **Danger (красный):** #DC2626 → #B91C1C (red-600 → red-700)
- **Neutral (серый):**
  - Фон страницы: #F8FAFC (slate-50)
  - Фон карточек: #FFFFFF (white)
  - Границы: #E2E8F0 (slate-200)
  - Текст основной: #0F172A (slate-900)
  - Текст вторичный: #64748B (slate-500)
  - Текст tertiary: #94A3B8 (slate-400)

```css
--shadow-md: 0 4px 6px -1px rgba(0, 0, 0, 0.1), 0 2px 4px -2px rgba(0, 0, 0, 0.1);
--shadow-lg: 0 10px 15px -3px rgba(0, 0, 0, 0.1), 0 4px 6px -4px rgba(0, 0, 0, 0.1);
--shadow-xl: 0 20px 25px -5px rgba(0, 0, 0, 0.1), 0 8px 10px -6px rgba(0, 0, 0, 0.1);
```

### 9. Анимации и переходы
- **Transition timing:** 200ms для hover, 300ms для сложных изменений
- **Easing:** ease-in-out для большинства, ease-out для появления элементов
- **Skeleton loaders:** вместо обычных спиннеров для карточек и списков
- **Page transitions:** fade-in при навигации между страницами
- **Micro-interactions:** ripple effect на кнопках, bounce на успешных действиях

### 10. Адаптивность
- **Breakpoints:**
  - Mobile: < 640px
  - Tablet: 640px - 1024px
  - Desktop: > 1024px
  
- **Mobile-first подход:**
  - Kanban board → вертикальная прокрутка вместо горизонтальной
  - Dashboard → 1 колонка вместо grid
  - Header → burger menu для навигации
  - Модальные окна → fullscreen на мобильных
